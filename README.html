Cloud Spend Forecasting with Monte Carlo Simulaton¶

Using FinOps FOCUS Data and Python’s PyMC to Create Probabilistic Budget Forecasts¶

Intro¶

This project demonstrates a simple approach to generate probabilistic budget forcasts using Python's PyMC package. By simulating a range of possible outcomes, Monte Carlo techniques enable Finance teams to consider uncertainty and variability in financial forecasts, providing more robust insights than the fixed-point estimates used in traditional budgeting.

Additionally, the project leverages the FOCUS dataset, a new standardized format for FinOps cloud spend data. Tracking and managing cloud computing costs has become critical as rapid growth in cloud services threaten to cause unallocated spending and budget overruns.

2. Historical Budget Variances¶

Since this is a toy problem, I will use made-up data for historical budget variances.

The simulated dataset below contains 6 years of historical budget variances for 2 cost drivers (API Management Services and Apache Spark pool). This historical data will be used to parameterize the prior distributions, which we will later sample from in the Monte Carlo simulation.By design,that the first cost drv(er, API Management Ser)vice, has historically accurate budgets, with all budget variances less than 1%. Conversely, the Apache cost driver had very inaccurate btudges with as much as a 50% unfavorable budget variance.

# Create a dictionary where the key is the cost driver name and the value is a list of historical budget variances
budget_names = budget_var_df['Cost Driver'].unique()

cost_variances = {}
for driver in budget_names:
    cost_variances[driver] = np.array([i+1 for i in budget_var_df[budget_var_df['Cost Driver']==driver]['Variance']])

Generating the Monte Carlo Simulation with PyMC¶

The Monte Carlo simulation below works in 3 steps:


Step 1:
For each cost driver, we must specify a prior distribution. The priors represent our best estimate of how likely we are to see different values of the driver. This could be based on expert opinion or past experience in similar cloud projects.

For the first 2 cost drivers (API Management and Apache Spark), I parameterize the prior distribution using the actual historical budget variances. I use a normal distribution centered at the mean budget variance and a standard deviation consistent with the historical deviation. These are different for the 2 drivers.

The rest of the cost drivers will come from a normal distribution with a mean equal to the actual cost and a standard deviation of 1. We can expect these to be fairly inaccurate.

Note: In the real world, a cost driver could be any variable that drives costs (e.g. resource hours, estimated monthly users, etc). But for simplicity in this example, I will assume that each unique FOCUS ResourceType is a cost driver.


Step 2:
With the prior distributions in hand, we then draw a large number of random samples for each cost driver.


Step 3:
Finally, we can compute the target costs of interest as a function of the cost drivers. For our basic example here, I will only compute “total costs” by doing a simple sum of all cost drivers. However, more complex cost functions can easily be modeled with the same setup.

# Generate Priors and Run the Monte Carlo Simulation

with pm.Model() as m:
    
    # Step 1: Generate priors for each Cost Driver
    for name, cost in zip(driver_names, YTD_billed_costs):
        # For the drivers with historical budget variance data, use the historical data to paramaterize the prior
        if name in budget_names:
            mu = (cost_variances[name]*cost).mean()
            sigma = (cost_variances[name]*cost).std()
            priors[name] = pm.Normal(name, mu=mu, sigma=sigma)
        # For the remaining drivers, use a generic parameterization
        else:
            sigma = 1
            mu = cost
            priors[name] = pm.Normal(name, mu=mu, sigma=sigma)

    # Step 3:  Calculate "Total Cost" as a function of the cost drivers
    total_cost = pm.Deterministic("Total Cost", np.sum([i for i in priors.values()]))
    
    
    # Step 2: Draw a large number of random samples
    trace = pm.sample(1000)

    # used to store samples
    ppc = pm.sample_posterior_predictive(trace)

Analyzing and Using the Results¶






















Python’s ArviZ package contains convenient functionality to visualize the sampled distributions. In the visualizations below, we can see sampled budget projections for each cost driver, as well as the calculated “Total Cost” (at the end).


As expected, the API Management Service cost has a very narrow range of budget projections because it has learned from historical budget variances that projections for the API service tend to be very accurate and reliable. Conversely, the Apache Spark driver has a much wider range of budget projections because it learned not to trust the expected cost.


The rest of the cost drivers are unreliable, since we put very little thought into their priors.
